# Canon and Revelation

Status expresses review state, while visibility expresses audience. New
records remain `idea` or `draft`. Only the Warden can approve a record as canon
or mark information revealed.

When players learn something, update the relevant clue, belief, revelation,
session, and world records without erasing the distinction between Warden truth
and player knowledge. Create a `handout` only for explicitly approved
player-facing material. Run validation before sharing and rebuild context after
approved canon changes.

Use `python scripts/drydock.py validate` before sharing and
`python scripts/drydock.py context` after approved canon changes.
