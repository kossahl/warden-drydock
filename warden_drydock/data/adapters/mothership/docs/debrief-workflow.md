# Debrief Workflow

After play, create `session` and `debrief` records with
`python scripts/drydock.py new`. Separate observed facts, character decisions,
discoveries, beliefs, injuries, Stress/Panic events, resource changes, and
proposed consequences. Create additional `consequence`, `clock`, or changed
entity records only when needed.

Keep the log and proposals in draft until the Warden confirms what occurred
and what becomes canon. Resolve contradictions explicitly. After approval, run
validation and rebuild context so the next session starts from reviewed state.
